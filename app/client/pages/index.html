Available vulnerabilities:

  • XML External Entity (XXE) Injection: This vulnerability occurs when an attacker can alter the XML file to access sensitive data or execute remote code. This is usually done by including an external entity in the XML input. The main cause of this vulnerability is the lack of proper input validation and/or a weak XML parser.
  • SQL Injection: This vulnerability occurs when an attacker can access the SQL database to access sensitive data. This is usually done by including malicious SQL commands in the input. The main cause of this vulnerability is the lack of proper input validation.
  • Clickjacking: Clickjacking is also known as UI dressing, where the attacker tricks users into clicking things that were hidden over visible objects. It can be done by embedding the target website in an iframe and making it invisible, or by simply turning a hidden object's opacity to 0. The goal of the user in this bug would be to identify where the clickjacking is happening and disable it before submitting the feedback.
